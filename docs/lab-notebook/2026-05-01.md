## SPI bring-up failed; switched to I2C

Initial bring-up attempted SPI (spi1 on Jetson 40-pin header pins 19/21/23/24).
WHO_AM_I read returned 0x00 / 0x80 / 0xC0 / random — MISO floating, sensor
not entering SPI mode. Verified:
- Power reaches sensor (responses change with bus activity)
- MOSI/MISO not swapped (continuity tests confirm SDA->Pin 19, SDO->Pin 21)
- SPI mode 3 set in code (per LSM6DSOX datasheet)
- Wiring continuity clean
Suspected cause: CS line not asserting at chip on this JetPack 6 + spidev
configuration, leaving sensor in default I2C mode at boot.

Diagnostic: rewired SDA/SCL to Jetson pins 3/5 (i2c8 -> /dev/i2c-7).
i2cdetect -y -r 7 shows device at 0x6a, the default LSM6DSOX I2C address.
Sensor confirmed alive and functional. Solder joints, power, and chip
all good. Failure was bus-specific, not sensor-specific.

Decision: switch to I2C for the rest of the project. Pre-registration
amendment to follow. SPI debug deferred indefinitely — not on critical
path for the latency comparison, which is bus-agnostic.

Tools available: resistors (kit on hand), Saleae Logic Pro 8 for analog
voltage measurement when needed (Tenma 72-8150 multimeter does not
support voltage measurement).
