### imu_logger truncation bug

During btest, last CSV row was truncated (3 fields instead of 4, no
trailing newline). Diagnosed: process killed before stdio buffer
flushed to disk. SIGTERM handler was missing — only SIGINT handled.

First fix attempt added signal(SIGTERM, on_sigint). Still produced
truncation at 30-sec duration but not at 5-sec, suggesting buffer-size
race rather than signal-handling race. Stopped chasing root cause.

Second fix added setvbuf(_IOLBF) for line-buffered output. Every row
flushes on \n. Defends against any abrupt termination (SIGTERM,
SIGKILL, power loss). Cost: one syscall per sample, negligible at
208 Hz on Orin Nano.

Both fixes committed together: 04957ae.

Verified end-to-end: btest with new logger produces clean trailing
newline, 4-field last row, "Stopped." cleanup message appears.
