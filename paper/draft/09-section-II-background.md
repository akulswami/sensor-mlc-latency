# §II. Background and Related Work

## II.A On-sensor inference in MEMS IMUs

Embedded machine learning in 6-axis MEMS IMUs is now a standard product feature. STMicroelectronics introduced the Machine Learning Core (MLC) in the LSM6DSOX [REF-AN5259], a configurable decision-tree engine running at the sensor's accelerometer ODR (configurable; 26 Hz in our deployment). The MLC consumes pre-computed features over sliding sample windows and emits class labels that the host reads via I²C bank-switched registers. The LSM6DSOX combines the MLC with a programmable Finite State Machine (FSM) for deductive algorithms; successor products such as the LSM6DSV16X (used in [REF-RAZMI-2026]) and LSM6DSO32X retain this MLC+FSM architecture with extended features and full-scale ranges. Competing vendors offer different approaches: Bosch's BHI260AP integrates a Fuser2 ARC EM4 32-bit RISC processor for programmable on-sensor classification, while Analog Devices' ADXL367 provides motion-triggered wake events at a coarser level of integration than full classification.

The consistent commercial messaging is that on-sensor inference moves work off the host, reducing data-bus traffic, host CPU utilization, and end-to-end latency. The first two claims are testable from system-level metrics; the third is testable only with wire-level instrumentation, which is rarely performed.

## II.B Wire-level latency in safety-critical edge ML

A growing class of edge-AI applications depends on bounded end-to-end latency from physical stimulus to actuation response. Wearable exoskeleton control [REF-RAZMI-2026] uses on-sensor MLC inference to detect locomotion modes (stance, level walking, stair ascent) for torque-assist control of a lower-limb exoskeleton; the architecture relies on the IMU's interrupt-driven event delivery to wake the microcontroller from low-power state, making the end-to-end stimulus-to-decision delay a system-level latency budget that the standard MLC read protocol must fit within. Industrial vibration monitoring with predictive maintenance uses MLC classifiers to gate alarms; alarm latency directly impacts response window. Healthcare wearables for arrhythmia or fall detection have application-defined response-time targets driven by clinical guidelines. In each domain, "the classification is correct" is a necessary but not sufficient property; "the classification reaches the actuator in time to act" is the safety-relevant property.

Vendor application notes characterize MLC architecture and per-class accuracy but do not report end-to-end stimulus-to-decision latency [REF-AN5259]. Independent characterizations focus on the host-side perspective — the latency from receiving the interrupt to acting on it — without measuring how long it took the silicon to fire that interrupt or how the host's read protocol contributes to the wire-level delay. The gap between functional correctness ("the classifier outputs the right label") and timing correctness ("the label reaches the host within budget, with the I²C read protocol's overhead and bus-contention behavior accounted for") is precisely the failure mode that conventional functional testing misses.

## II.C Pre-registration in sensor measurement

Pre-registration — the public, externally-timestamped specification of hypotheses, exclusion criteria, statistical tests, and multiplicity-correction strategies before data collection — is established practice in clinical trials and increasingly common in psychology and machine learning [REF-NOSEK-2018]. Its application to sensor measurement is rare. The contribution we make is methodological as well as empirical: a pre-registered measurement campaign with externally-timestamped Zenodo DOIs provides a chain of custody that distinguishes pre-specified hypotheses from post-hoc rationalizations, and a record of methodological changes (with their dates and motivating evidence) that an auditor can verify against the public repository.

The pre-registration discipline does not prevent honest scientific surprise. Three of our originally pre-registered hypotheses were falsified during the study, including the headline directional claim that "on-sensor inference is faster than host-side inference." Each falsification is preserved on the public record with its motivating evidence and an external Zenodo timestamp. The framework's value is not that the original hypotheses survived but that the falsifications are documentably honest.
