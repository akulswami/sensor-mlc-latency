# §II. Background and Related Work

## II.A On-sensor inference in MEMS IMUs

Embedded machine learning in 6-axis MEMS IMUs is now a standard product feature. STMicroelectronics introduced the Machine Learning Core (MLC) in the LSM6DSOX [REF-AN5259], a configurable decision-tree engine running at the accelerometer ODR (26 Hz in our deployment) that consumes features over sliding sample windows and emits class labels the host reads via I²C bank-switched registers. Successor parts (LSM6DSV16X [REF-RAZMI-2026], LSM6DSO32X) retain this architecture, and competing vendors offer analogues (Bosch BHI260AP, Analog Devices ADXL367). The consistent commercial messaging is that on-sensor inference reduces data-bus traffic, host CPU utilization, and end-to-end latency. The first two claims are testable from system-level metrics; the third is testable only with wire-level instrumentation, which is rarely performed.

## II.B Wire-level latency in safety-critical edge ML

A growing class of edge-AI applications depends on bounded stimulus-to-actuation latency. Wearable exoskeleton control [REF-RAZMI-2026] uses on-sensor MLC inference for torque-assist gait control; industrial predictive maintenance gates alarms on MLC classifiers; healthcare wearables for arrhythmia or fall detection carry clinically-driven response-time targets. In each domain, "the classification is correct" is necessary but not sufficient — "the classification reaches the actuator in time" is the safety-relevant property. Yet vendor application notes characterize MLC architecture and per-class accuracy without reporting end-to-end stimulus-to-decision latency [REF-AN5259], and independent characterizations measure the host-side interrupt-to-action latency without measuring how the read protocol contributes to the wire-level delay. That gap — between functional and timing correctness — is the failure mode conventional functional testing misses.

## II.C Pre-registration in sensor measurement

Pre-registration — public, externally-timestamped specification of hypotheses, exclusion criteria, tests, and multiplicity correction before data collection — is established in clinical trials and increasingly common in psychology and machine learning [REF-NOSEK-2018], but rare in sensor measurement. Externally-timestamped Zenodo DOIs provide a chain of custody distinguishing pre-specified hypotheses from post-hoc rationalization. The discipline does not prevent honest surprise: three originally pre-registered hypotheses were falsified during this study, including the headline directional claim that on-sensor inference is faster than host-side inference. Each falsification is preserved on the public record with its motivating evidence. The framework's value is not that the original hypotheses survived, but that the falsifications are documentably honest.
