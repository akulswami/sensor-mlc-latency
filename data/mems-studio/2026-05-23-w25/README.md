# MEMS Studio artifacts — 2026-05-23, window=25

This directory captures the artifacts MEMS Studio 2.3.1 produced
during the w=25 candidate decision-tree training run, per the spec's
three-window evaluation requirement (`docs/training-data-spec.md`
§"Window length"). One of the three candidate windows alongside w=75
(`data/mems-studio/2026-05-22-w75/`) and w=200
(`data/mems-studio/2026-05-23-w200/`).

**Hyperparameters in this run** (see mlc_settings.json for full
config):

- Window length: **25 samples** (240 ms @ 104 Hz MLC ODR)
- MLC ODR: **104 Hz** (cap per AN5259, locked by v4 amendment)
- Accel ODR: 208 Hz, full scale 2 g
- Filter: IIR1_Acc_V, HP at fc=1 Hz / fs=104 Hz
  (b1=0.970669, b2=-0.970669, a2=+0.941339, MEMS Studio's
  Convention B sign for a2)
- Features: VARIANCE and PEAK_TO_PEAK on Acc_V_filter_1
- Training data: sessions 1 (2026-05-20) + 2 (2026-05-21) only
  (session 3 unused per v5 amendment; sessions 4-prime/5/6 are §9
  test sessions, not training)
- Tree algo: new train, max_depth=2047, min_leaf=30, holdout=0.20
- Class codes: still=0, motion=4

## Trained tree (see ST_decision_tree.txt)

```
F2_ABS_PEAK_TO_PEAK_ACC_V_FILTER_1 <= 0.028564 : still (8800/711.0)
F2_ABS_PEAK_TO_PEAK_ACC_V_FILTER_1  > 0.028564 : motion (7441/35.0)
```

Single split on p2p_norm, depth 1. Same feature as w=75 (which also
picked p2p) but threshold is 0.58× w=75's (0.028564 vs 0.049316) —
expected because shorter windows of the same signal yield smaller p2p.

**Training-set accuracy: 95.42%** (Cohen's kappa 90.85%, 929 of 20,302
misclassified). The misclassifications are highly asymmetric:

| Actual ↓ / Predicted → | still | motion |
|---|---|---|
| still  | 10,097 | 54  |
| motion | **875** | 9,276 |

875 motion windows classify as still (motion recall 0.91); only 54
still windows misclassify as motion. Physical interpretation: a 240 ms
window can land entirely on the low-velocity / zero-crossing phase of
the servo's 0°↔150° oscillation, in which case the acceleration p2p
over those samples falls below the threshold even though the servo is
moving. Longer windows (w=75 = 721 ms, w=200 = 1.92 s) span enough of
a full oscillation to always catch a high-velocity phase, so this
failure mode disappears at longer windows.

w=25's depth-1 tree passes the spec's depth ≤5 ceiling. Whether it
also clears the §9 accuracy gate (≥90% absolute, ≤2pp host-silicon
gap) is determined by the v5-pre-registered S5 capture session, not
yet collected.

## MEMS Studio quirks (carried from 2026-05-22-w75/README.md, plus new)

Carried from w=75:

1. **Convention B for filter coefficients.** MEMS Studio's UI uses
   H(z) = (b1 + b2 z^-1) / (1 - a2 z^-1), opposite sign on a2 vs
   AN5259. `mlc_json_to_parity.py` sign-flips a2 when emitting
   tree.json.
2. **CRLF line endings in features.arff.** Tooling handles \r\n.
3. **ARFF file path field doesn't accept manual edits via the file
   picker.** The "Browse" dialog only allows selecting existing
   files; the text field beneath it can be edited directly, but in
   practice it's simpler to let MEMS Studio write to its default
   `/home/akulswami/features.arff` and `mv`/`cp` the artifact
   afterward.

New in this run:

4. **MEMS Studio cross-step file-path coupling.** After
   `mv`'ing the generated ARFF out of MEMS Studio's default location,
   the Decision tree generation step failed with "Unable to open file:
   /home/akulswami/features.arff". MEMS Studio holds the ARFF input
   path generated at ARFF-generation time and reads from there at
   tree-generation time. Workaround: leave artifacts at default
   `/home/akulswami/` paths until all MEMS Studio steps (ARFF →
   Tree → Config → Export Header) complete, then `mv` everything to
   the per-window directory in one batch.
5. **Impure-leaf count format `(N/M.M)` in tree-text dump.** When the
   trained tree has leaves with misclassifications (as w=25 does),
   MEMS Studio writes leaf counts as `(N/M.M)` where M is the count
   of misclassified samples in that leaf. The original
   `mlc_json_to_parity.py` regex only matched the pure-leaf `(N)`
   form; patched in commit c8ed4e8 to accept both forms.
