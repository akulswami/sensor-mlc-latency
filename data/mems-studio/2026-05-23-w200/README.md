# MEMS Studio artifacts — 2026-05-23, window=200

This directory captures the artifacts MEMS Studio 2.3.1 produced
during the w=200 candidate decision-tree training run, per the spec's
three-window evaluation requirement (`docs/training-data-spec.md`
§"Window length"). One of the three candidate windows alongside w=25
(`data/mems-studio/2026-05-23-w25/`) and w=75
(`data/mems-studio/2026-05-22-w75/`).

**Hyperparameters in this run** (see mlc_settings.json for full
config):

- Window length: **200 samples** (1.923 s @ 104 Hz MLC ODR)
- MLC ODR: **104 Hz** (cap per AN5259, locked by v4 amendment)
- Accel ODR: 208 Hz, full scale 2 g
- Filter: IIR1_Acc_V, HP at fc=1 Hz / fs=104 Hz
  (b1=0.970669, b2=-0.970669, a2=+0.941339, MEMS Studio's
  Convention B sign for a2)
- Features: VARIANCE and PEAK_TO_PEAK on Acc_V_filter_1
- Training data: sessions 1 (2026-05-20) + 2 (2026-05-21) only
  (session 3 unused per v5 amendment; sessions 4-prime/5/6 are §9
  test sessions, not training)
- Tree algo: new train, max_depth=2047, min_leaf=30, holdout=0.20
- Class codes: still=0, motion=4

## Trained tree (see ST_decision_tree.txt)

```
F1_ABS_VARIANCE_ACC_V_FILTER_1 <= 0.002094 : still (996)
F1_ABS_VARIANCE_ACC_V_FILTER_1  > 0.002094 : motion (984)
```

Single split, depth 1, **on the VARIANCE feature (not PEAK_TO_PEAK)**.
This is the only one of the three candidates whose tree picked variance
over p2p. Physical interpretation: at 1.9-second windows, the servo's
0°↔150° oscillation completes multiple full cycles within a window, and
variance accumulates as the time-integral of the squared signal — for
sustained oscillation this scales linearly with window length, making
variance a clean discriminator. PEAK_TO_PEAK saturates once a window
covers one full oscillation cycle, so it gains nothing from a 1.9 s
window over a ~360 ms one (w=75) where it already separated cleanly.
MEMS Studio's tree training picked the feature with the cleaner split
on each window length.

**Training-set accuracy: 100.00%** (Cohen's kappa 100%). Pure leaves
(no impurity slash in leaf counts). Confusion matrix is perfectly
diagonal: 1238/1238 still correct, 1238/1238 motion correct.

w=200's depth-1 tree passes the spec's depth ≤5 ceiling. Whether it
also clears the §9 accuracy gate (≥90% absolute, ≤2pp host-silicon
gap) is determined by the v5-pre-registered S6 capture session, not
yet collected.

## Note on window-length latency tradeoff

w=200 must accumulate 200 MLC samples (1.923 s @ 104 Hz MLC ODR)
before its first classification. This is a fundamental floor on
first-decision latency for w=200, independent of host vs silicon
inference speed. For comparison, w=75 first-decision latency is
~721 ms, w=25 is ~240 ms. Since this paper's primary outcome is
inference latency under stress (§6), the window-length selection
implicitly trades off classifier accuracy against first-decision
latency. If two windows tie on §9 accuracy (as w=75 and w=200
appear to on training data), w=75 wins on first-decision latency.
This tradeoff is not pre-registered as a selection criterion; the
spec's selection is on validation accuracy with depth penalty.
Documenting the tradeoff here for transparency.
